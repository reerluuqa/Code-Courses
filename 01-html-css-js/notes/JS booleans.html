What Are Booleans?Mission 1

A Boolean is the simplest data type in programming ‚Äî it can only be one of two values: true or false. Think of it like a light switch: on or off, yes or no.

let isLoggedIn = true; let isAdmin = false; // A Boolean answers a yes/no question: // "Is the user logged in?" ‚Üí true // "Is the user an admin?" ‚Üí false

Every if statement, every toggle, every checkbox ‚Äî they all rely on Booleans under the hood.

Comparison OperatorsMission 2

Comparisons produce Booleans. You ask JavaScript "is 5 greater than 3?" and it answers true or false.

// Strict equality & inequality 5 === 5 // true (same value AND same type) 5 === "5" // false (number vs string!) 5 !== 3 // true (not equal) // Greater / Less 10 > 7 // true 4 < 2 // false 5 >= 5 // true (greater OR equal) 3 <= 1 // false

Use === (strict) instead of == (loose). Strict checks both value AND type, preventing sneaky bugs.

Truthy & FalsyMission 4

JavaScript treats some non-Boolean values as false ("falsy") and everything else as true ("truthy") when used in conditions.

// The 6 FALSY values ‚Äî memorize these! false // the Boolean false itself 0 // the number zero "" // empty string null // intentional "nothing" undefined // variable declared but not set NaN // "Not a Number" // EVERYTHING else is TRUTHY: "hello" ‚Üí truthy 42 ‚Üí truthy [] ‚Üí truthy {} ‚Üí truthy "0" ‚Üí truthy (it's a non-empty string!)

That's why if("hello") runs ‚Äî the string "hello" is truthy!

Double Bang !!Mission 6

!! converts any value to its true Boolean equivalent ‚Äî the first ! flips it, the second ! flips it back.

// Step-by-step: convert "hello" to Boolean !"hello" // "hello" is truthy ‚Üí NOT truthy ‚Üí false !!"hello" // NOT false ‚Üí true ‚úì !!"" // false (empty string is falsy) !!0 // false !!null // false !!42 // true !![] // true (empty array is truthy!) // Same as using Boolean(): Boolean("hello") // true Boolean(0) // false

You'll see !! often in real code. It's a quick way to check if a variable has a "real" value.
