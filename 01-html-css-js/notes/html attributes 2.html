What Is an Attribute?

ü§î The Simplest Explanation

Think of HTML elements like people. Every person has a name, but they also have characteristics ‚Äî like hair color, height, age, and so on.

Attributes are the characteristics of HTML elements. They give extra information or instructions to the element, telling the browser how to display or behave.

üí° Real-World Analogy

üë§ A Person

  • Name: "Sarah"
  • Hair Color: "brown" ‚Üê characteristic
  • Height: "tall" ‚Üê characteristic
  • Age: "25" ‚Üê characteristic

üñºÔ∏è An HTML Image

  • Tag: <img>
  • src: "photo.jpg" ‚Üê attribute!
  • width: "300" ‚Üê attribute!
  • alt: "A cute cat" ‚Üê attribute!
HTML
<!-- WITHOUT attributes ‚Äî broken! The browser doesn't know WHAT to show --> <img> <!-- WITH attributes ‚Äî works! Now the browser knows everything it needs --> <img src="cat.jpg" alt="A cute cat" width="300">
üìå Key Idea

Without attributes, many HTML elements are like giving someone an empty envelope ‚Äî there's nothing useful inside!

What Is a Boolean Attribute?

üîò Regular vs Boolean Attributes

A regular attribute has a name AND a value: src="photo.jpg"

A boolean attribute only needs its NAME to work. Its mere presence means "yes / true / on", and its absence means "no / false / off".

üí° The Light Switch Analogy

Think of a Boolean Attribute like a light switch on a wall:

ON
Switch EXISTS = Light ON
<input disabled>
OFF
Switch REMOVED = Light OFF
<input>

You DON'T write switch="on" ‚Äî just having the switch there IS "on"!

HTML ‚Äî All valid ways to write boolean attributes
<!-- ‚úÖ ALL of these are CORRECT and mean the same thing: --> <input type="text" disabled> <!-- ‚Üê RECOMMENDED (cleanest) --> <input type="text" disabled=""> <input type="text" disabled="disabled">
üö® THE #1 BOOLEAN ATTRIBUTE TRAP

disabled="false" ‚Üí STILL DISABLED! (attribute is present!)

disabled="no" ‚Üí STILL DISABLED! (attribute is present!)

disabled="0" ‚Üí STILL DISABLED! (attribute is present!)

To turn it OFF, you must REMOVE the attribute entirely:

<input type="text"> ‚Üí NOW it's enabled (no disabled word present)

"Is the word 'disabled' anywhere in the tag? YES ‚Üí disabled. NO ‚Üí not disabled. The VALUE doesn't matter!"

Common Mistakes & Pitfalls to Avoid

1

‚ùå Wrong
<img src=my photo.jpg>
Browser thinks src = "my"
and "photo.jpg" = ??? ERROR!
‚úÖ Correct
<img src="my photo.jpg">
Quotes keep the value together!
2

Putting Attributes in the Closing Tag

‚ùå Wrong

‚úÖ Correct
<p class="greeting">Hello</p>

‚ùå Wrong ‚Äî Same id twice!

<h2 id="title">Second</h2>

‚úÖ Remove it entirely!

Forgetting alt on Images

‚ùå Missing or bad alt

‚ùå Server receives nothing!

Not Connecting label and input
