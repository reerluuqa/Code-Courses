Today’s lesson: <aside> means “related but not main”

This single-page “mini app” shows two versions of the same layout. One uses semantic HTML (main, article, aside); the other uses generic divs. They can look identical — but they do not mean the same thing.

1) Two pages that look similar (but mean different things)

Imagine a blog page. The articles are the “main meal”. The sidebar (popular posts, ads, related links) is like a “side dish”. <aside> is how you tell the browser and assistive tech: “this is a side dish”.

2) What a computer learns from semantic tags

A human can glance and see “that’s a sidebar”. A computer needs explicit labels. Semantic tags provide those labels.

Key idea If you can delete it and the page still makes sense, it’s often an <aside>.
Example Articles = main story. “Popular posts” = helpful extra, so it belongs in <aside>.

3) Tiny “meaning map” (read like a robot)

4) Your exercises (do these today)

These are beginner-friendly. You can do them by editing this file and refreshing your browser.

  1. Delete test: In Example A, imagine removing the <aside>. Does the page still function as “a blog with posts”? If yes, that supports the idea that the removed section was supporting.
  2. Move test: In the semantic example, move the <aside> above <main> in the HTML. The meaning is still “supporting content” even if it appears earlier on the page.
  3. Make an <aside> inside an <article>: Add a mini aside under the first article like “Definitions” or “Quick tip”. That’s still valid if it’s supporting that specific article.
  4. Wrong usage challenge: Put the actual article text inside <aside> and the “Popular posts” in <main>. Ask yourself: what meaning did you accidentally communicate?
  5. Labeling habit: Add aria-label to the sidebar aside: <aside aria-label="Popular posts">. It makes the landmark clearer.
Rule of thumb Use <main> for the page’s purpose; use <aside> for related extras.

5) Quick quiz (answer in your head)

  • If a page is a recipe, where would “ingredient substitutions” go?
  • If a page is a news article, where would “related stories” go?
  • If a page is a product page, where would “customers also bought” go?

In most cases: those are <aside> because they’re helpful, but not the main thing.
