ğŸ­ Divs in Modern Web Development

When to USE <div>:

  • Styling containers - When you need a box just for CSS/JS
  • Layout wrappers - Grid/flex containers
  • JavaScript hooks - Elements you'll manipulate with JS
  • No semantic meaning exists - Pure visual grouping
Good Div Usage:
<div class="container">
<div class="grid">
<div class="modal">
<div class="tooltip">

When to AVOID <div>:

  • Navigation â†’ Use <nav>
  • Head/Foot sections â†’ Use <header>/<footer>
  • Main content â†’ Use <main>
  • Articles/sections â†’ Use <article>/<section>
Bad Div Usage:
<div id="navigation"> â† Should be <nav>
<div class="header"> â† Should be <header>
<div id="main-content"> â† Should be <main>
ğŸ’¡ The Reality: Most modern websites use a combination of semantic elements AND divs. Semantic elements for structure + meaning, divs for layout + styling.

â™¿ Accessibility Essentials

ğŸ¯ MUST-HAVE Accessibility Features

ARIA Attributes

  • aria-label="description" - Direct label
  • aria-labelledby="id" - Reference to labeling element
  • aria-describedby="id" - Reference to description
  • role="navigation" - Reinforces element purpose

Keyboard Navigation

  • Skip links for keyboard users
  • Logical tab order
  • Focus indicators
  • Proper heading hierarchy

ğŸ“ Accessibility Code Examples

<!-- Screen reader label for navigation -->

<!-- Proper heading structure -->

  <h2>Section Title</h2>
    <h3>Subsection Title</h3>

<!-- Machine-readable date -->

ğŸ’¡ Golden Rule: Use native semantic elements first, then add ARIA only when necessary. Don't use ARIA to fix bad HTML!
