Core Concepts

What is a TAG? üè∑Ô∏è

A tag is a special instruction that tells the browser what to do with your content. Think of it like a label on a container.

Tags are written inside angle brackets: < >

<p>This text is inside a paragraph tag</p>
<h1>This is a heading</h1>

Tip: Most tags come in pairs - an opening tag and a closing tag. The closing tag has a forward slash /.

What is an ELEMENT? üß±

An element is the complete package: opening tag + content + closing tag.

If a tag is like a lunchbox label saying "sandwich inside", the element is the entire lunchbox with the sandwich.

<div class="my-box">
    This is the content inside the div element
</div>

1. Always close your tags (with some exceptions like <img>)

2. Nest elements properly (don't overlap them)

3. Use the right element for the job (headings for titles, paragraphs for text)

Common Mistakes to Avoid

Forgetting to Close Tags

One of the most common mistakes beginners make is forgetting the closing tag.

Wrong
<p>This paragraph has no closing tag
<p>This is another paragraph</p>

The browser doesn't know where the first paragraph ends!

Right
<p>This paragraph is properly closed</p>
<p>This is another paragraph</p>

Every opening tag has a matching closing tag.

Incorrect Nesting (Overlapping Tags)

Tags should be nested like Russian dolls, not overlapping.

Wrong
<strong><p>Bold paragraph</strong></p>

The tags are overlapping instead of nesting properly.

Right
<p><strong>Bold paragraph</strong></p>

The strong tag is completely inside the p tag.

Missing Alt Text for Images

The alt attribute describes images for screen readers and when images don't load.

Wrong
<img src="cat.jpg">

No description for people using screen readers or if the image fails to load.

Right
<img src="cat.jpg" alt="A fluffy orange cat sleeping on a windowsill">

Screen readers will read the alt text, and it shows if the image doesn't load.
