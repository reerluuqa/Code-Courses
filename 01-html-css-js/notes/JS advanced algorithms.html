What is an Algorithm?

An algorithm is a step-by-step recipe to solve a problem. A data structure is how you organize information in memory. Together, they determine if your code runs in 1 millisecond or 10 hours.

üí° The Core Question: "If I double the input size, how much slower does my code get?" This is what interviewers are REALLY asking.
‚ùå Bad Code: O(n¬≤) ‚Äî 1,000 items takes 1M operations. 1,000,000 items takes 1 TRILLION.
‚úÖ Good Code: O(n log n) ‚Äî 1,000 items takes 10,000 ops. 1,000,000 takes 20,000,000.
üèõÔ∏è
The Mental Model: 3 Questions to Ask Every Time
1. What is the TIME complexity?
How many operations as input grows?
2. What is the SPACE complexity?
How much extra memory do I need?
3. Can I do better?
Trade time for space or vice versa?

// WHY IS BINARY SEARCH O(log n)? // Each step HALVES the search space // n=16: step1‚Üí8, step2‚Üí4, step3‚Üí2, step4‚Üí1 // 4 steps for 16 elements = log‚ÇÇ(16) = 4 ‚úì function binarySearch(arr, target) { let lo = 0, hi = arr.length - 1; while(lo <= hi) { const mid = Math.floor((lo + hi) / 2); if(arr[mid] === target) return mid; if(arr[mid] < target) lo = mid + 1; // eliminate left half else hi = mid - 1; // eliminate right half } return -1; } // O(log n) time, O(1) space ‚Äî very efficient!

Use when: You need indexed access, you know the size, or you're iterating everything. It's your default choice.

Stack (LIFO)

Last In, First Out. Like a plate stack. Used for: undo, DFS, call stack, balanced parentheses.
// Use JS Array as Stack const stack = []; stack.push(1); // [1] O(1) stack.push(2); // [1,2] O(1) stack.pop(); // ‚Üí 2 O(1) stack.at(-1); // peek O(1) // CLASSIC: Valid Parentheses function isValid(s) { const st = []; const map = {')':'(', '}':'{', ']':'['}; for(let c of s) { if('([{'.includes(c)) st.push(c); else if(st.pop() !== map[c]) return false; } return st.length === 0; }

First In, First Out. Like a line. Used for: BFS, task scheduling, print queues.
// Simple Queue with Array const queue = []; queue.push(1); // enqueue O(1) queue.push(2); // enqueue O(1) queue.shift(); // dequeue O(n)! ‚ö†Ô∏è // Better: Track head index class Queue { constructor() { this.data = {}; this.head = 0; this.tail = 0; } enqueue(v) { this.data[this.tail++] = v; } dequeue() { const v = this.data[this.head]; delete this.data[this.head++]; return v; // O(1)! ‚úì } size() { return this.tail - this.head; } }

BST Property: left < parent < right
// ‚ïê‚ïê‚ïê TREE TRAVERSALS (memorize all 4!) ‚ïê‚ïê‚ïê class TreeNode { constructor(val) { this.val=val; this.left=null; this.right=null; } } // DFS ‚Äî Inorder (Left‚ÜíRoot‚ÜíRight) = SORTED for BST function inorder(node, result=[]) { if(!node) return result; inorder(node.left, result); result.push(node.val); inorder(node.right, result); return result; } // BFS ‚Äî Level Order (very common in interviews!) function levelOrder(root) { if(!root) return []; const result=[], queue=[root]; while(queue.length) { const size = queue.length; const level = []; for(let i=0; i<size; i++) { const node = queue.shift(); level.push(node.val); if(node.left) queue.push(node.left); if(node.right) queue.push(node.right); } result.push(level); } return result; } // Max Depth function maxDepth(root) { if(!root) return 0; return 1 + Math.max(maxDepth(root.left), maxDepth(root.right)); }

Heap (Priority Queue)
A heap is a complete binary tree where parent is always greater (max-heap) or smaller (min-heap) than children. Always O(log n) insertions, O(1) peek of min/max.
// JS doesn't have built-in heap, implement one! class MinHeap { constructor() { this.data = []; } push(val) { this.data.push(val); this._bubbleUp(); } pop() { const min = this.data[0]; const last = this.data.pop(); if(this.data.length) { this.data[0] = last; this._sinkDown(); } return min; } peek() { return this.data[0]; } // O(1)! size() { return this.data.length; } _bubbleUp() { /* O(log n) */ } _sinkDown() { /* O(log n) */ } } // USE CASE: K Largest Elements ‚Üí use min-heap of size K! // USE CASE: Merge K sorted lists // USE CASE: Dijkstra's shortest path // USE CASE: Median of data stream

Algorithm Implementations
// MERGE SORT ‚Äî Divide & Conquer ‚Üí O(n log n) function mergeSort(arr) { if(arr.length <= 1) return arr; // base case const mid = Math.floor(arr.length / 2); const left = mergeSort(arr.slice(0, mid)); // sort left half const right = mergeSort(arr.slice(mid)); // sort right half return merge(left, right); } function merge(left, right) { const result = []; let i=0, j=0; while(i < left.length && j < right.length) { if(left[i] <= right[j]) result.push(left[i++]); else result.push(right[j++]); } return [...result, ...left.slice(i), ...right.slice(j)]; } // ANALYSIS: // Log n levels of recursion // Each level does O(n) work in merge // Total: O(n log n) ‚Äî guaranteed! // Space: O(n) ‚Äî new arrays at each level
// QUICKSORT ‚Äî Partition around pivot ‚Üí O(n log n) avg function quickSort(arr, lo=0, hi=arr.length-1) { if(lo >= hi) return arr; const pivotIdx = partition(arr, lo, hi); quickSort(arr, lo, pivotIdx - 1); // sort left quickSort(arr, pivotIdx + 1, hi); // sort right return arr; } function partition(arr, lo, hi) { const pivot = arr[hi]; // pick last as pivot let i = lo - 1; for(let j=lo; j<hi; j++) { if(arr[j] <= pivot) { i++; [arr[i], arr[j]] = [arr[j], arr[i]]; // swap } } [arr[i+1], arr[hi]] = [arr[hi], arr[i+1]]; return i + 1; } // ‚ö†Ô∏è WORST CASE: O(n¬≤) when pivot is always min/max // FIX: Random pivot or median-of-three // Space: O(log n) ‚Äî recursion depth
// BINARY SEARCH PATTERNS // Classic ‚Äî find exact value function binarySearch(arr, target) { let lo=0, hi=arr.length-1; while(lo<=hi) { const mid=Math.floor((lo+hi)/2); if(arr[mid]===target) return mid; arr[mid]<target ? lo=mid+1 : hi=mid-1; } return -1; } // Find leftmost position ‚Äî template function leftBound(arr, target) { let lo=0, hi=arr.length; while(lo<hi) { const mid=Math.floor((lo+hi)/2); arr[mid]<target ? lo=mid+1 : hi=mid; } return lo; } // Binary search on ANSWER ‚Äî advanced pattern! // "Find minimum X such that condition(X) is true" function binaryOnAnswer(lo, hi, condition) { while(lo<hi) { const mid=Math.floor((lo+hi)/2); condition(mid) ? hi=mid : lo=mid+1; } return lo; }
// DYNAMIC PROGRAMMING ‚Äî "Intelligent recursion" // Key insight: cache subproblem results! // ‚ïê‚ïê‚ïê TOP-DOWN (Memoization) ‚ïê‚ïê‚ïê function climbStairs(n, memo={}) { if(n in memo) return memo[n]; if(n<=2) return n; memo[n] = climbStairs(n-1,memo) + climbStairs(n-2,memo); return memo[n]; } // ‚ïê‚ïê‚ïê BOTTOM-UP (Tabulation) ‚ïê‚ïê‚ïê function climbStairsDP(n) { const dp = new Array(n+1).fill(0); dp[1]=1; dp[2]=2; for(let i=3; i<=n; i++) { dp[i] = dp[i-1] + dp[i-2]; } return dp[n]; } // ‚ïê‚ïê‚ïê SPACE OPTIMIZED ‚Äî only keep what you need ‚ïê‚ïê‚ïê function climbStairsOpt(n) { let a=1, b=2; for(let i=3; i<=n; i++) { [a, b] = [b, a+b]; } return b; // O(n) time, O(1) space! } // CLASSIC DP PROBLEMS: // Knapsack, LCS, Edit Distance, Coin Change // Longest Increasing Subsequence

Two Pointers
Arrays & Strings

Use two indices that move toward each other (or both forward). Eliminates the need for nested loops in many cases.

‚úÖ Problems: Two sum (sorted), remove duplicates, palindrome check, container with most water
// Pair that sums to target (sorted array) function twoSum(arr, target) { let l=0, r=arr.length-1; while(l<r) { const sum = arr[l]+arr[r]; if(sum===target) return [l,r]; sum<target ? l++ : r--; } return []; } // O(n) vs O(n¬≤) brute force!
ü™ü
Sliding Window
Subarray/Substring Problems

Maintain a window of elements that slides through the array. Avoids recalculating from scratch each time.

‚úÖ Problems: Max sum subarray, longest substring without repeating chars, minimum window substring
// Max sum of K consecutive elements function maxSum(arr, k) { let sum = arr.slice(0,k).reduce((a,b)=>a+b); let max = sum; for(let i=k; i<arr.length; i++) { sum += arr[i] - arr[i-k]; // slide: add new, remove old max = Math.max(max, sum); } return max; } // O(n) vs O(n*k) brute force! // VARIABLE SIZE WINDOW function longestUniqueSubstr(s) { const seen = new Map(); let max=0, left=0; for(let r=0; r<s.length; r++) { if(seen.has(s[r])) left=Math.max(left,seen.get(s[r])+1); seen.set(s[r],r); max=Math.max(max,r-left+1); } return max; }

Fast & Slow Pointers
Linked Lists & Cycles

Two pointers at different speeds. Fast moves 2x, slow moves 1x. They meet at cycle start or middle.

// Find cycle in linked list / detect middle function findCycleStart(head) { let slow=head, fast=head; while(fast&&fast.next) { slow=slow.next; fast=fast.next.next; if(slow===fast) { // cycle detected! slow=head; // reset slow to head while(slow!==fast) { slow=slow.next; fast=fast.next; } return slow; // cycle start! } } return null; }

‚ö†Ô∏è Mistake 1: Using Array.includes() in a Loop

Calling includes() or indexOf() inside a loop secretly creates O(n¬≤) complexity.

‚ùå BAD ‚Äî O(n¬≤)
for(let x of arr1) { // includes = O(n) inside O(n)! if(arr2.includes(x)) { result.push(x); } }
‚úÖ GOOD ‚Äî O(n)
const set2 = new Set(arr2); for(let x of arr1) { // Set.has = O(1)! if(set2.has(x)) { result.push(x); } }
‚ö†Ô∏è Mistake 2: Forgetting to Handle Edge Cases

Interviewers always test empty input, single elements, and negative numbers. Always ask/handle these!

// ‚ùå Fails on empty array function maxBad(arr) { let max = arr[0]; for(let i=1; i<arr.length; i++) ... } // ‚úÖ Always guard against edge cases function maxGood(arr) { if(!arr || arr.length === 0) return null; let max = -Infinity; // handles negatives! for(let n of arr) max = Math.max(max, n); return max; }

‚ö†Ô∏è Mistake 5: Using == Instead of ===
0 == '' // true ‚Üê DANGEROUS! 0 === '' // false ‚Üê CORRECT null == undefined // true ‚Üê might cause bugs false == 0 // true ‚Üê shocking! // ‚úÖ ALWAYS use === in algorithms // Only use == when you explicitly want type coercion
‚ö†Ô∏è Mistake 6: Off-By-One Errors in Binary Search
// ‚ùå Common mistakes: while(lo < hi) // vs lo <= hi hi = mid // vs hi = mid - 1 lo = mid // vs lo = mid + 1 // ‚úÖ The safe template ‚Äî understand each line! let lo=0, hi=arr.length-1; while(lo <= hi) { // ‚Üê <= to check single element const mid = Math.floor((lo+hi)/2); // ‚Üë Use floor, not ceil (for even lengths) if(arr[mid]===target) return mid; if(arr[mid]<target) lo=mid+1; // ‚Üê +1 to skip mid else hi=mid-1; // ‚Üê -1 to skip mid }
‚ö†Ô∏è Mistake 7: Forgetting the visited Set in Graph Traversal

Without tracking visited nodes, you get infinite loops!

‚ùå INFINITE LOOP
function dfs(node) { console.log(node); for(let n of graph[node]) { dfs(n); // revisits! } }
‚úÖ SAFE
const visited = new Set(); function dfs(node) { if(visited.has(node)) return; visited.add(node); for(let n of graph[node]) { dfs(n); } }
