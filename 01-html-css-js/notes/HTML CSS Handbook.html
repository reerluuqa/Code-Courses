Section 01

HTML ‚Äî Core Fundamentals

HTML is the skeleton of every web page. Before you can style anything, you must understand how to structure content with semantic, well-organized markup.

1.1 What is HTML

HTML (HyperText Markup Language) is a declarative language that tells the browser what content exists and how that content is organized into a document. It is not a programming language ‚Äî it has no variables, loops, or logic. Instead, it uses tags to annotate text, images, links, and other media so the browser can render them correctly.

Key Principles

  • Top-to-bottom parsing: Browsers read HTML sequentially from the first line to the last. Elements that appear first in the source are rendered first on the page.
  • Nesting: Elements can contain other elements, forming a tree-like structure called the DOM (Document Object Model).
  • Semantic meaning: Tags carry meaning. A <header> is not the same as a <div>, even if they visually look identical. Semantics affect accessibility, SEO, and maintainability.

The Anatomy of an HTML Document

<!-- Every HTML page needs this boilerplate --> <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Page Title</title> </head> <body> <!-- All visible content goes here --> <h1>Hello World</h1> <p>This is a paragraph.</p> </body> </html>HTML
üîë Key Concept
<!DOCTYPE html> is not a tag ‚Äî it's a declaration that tells the browser to use modern HTML5 standards. Without it, the browser enters "quirks mode" and rendering becomes unpredictable.

1.2 Tags & Elements

A tag is the markup syntax ‚Äî the angle brackets and the name (e.g., <p>). An element is the complete unit: the opening tag + content + closing tag, plus any attributes.

Normal Elements (Have Closing Tags)

1.6 Spacing & Indentation

HTML collapses all whitespace. Multiple spaces, tabs, and line breaks in your source code are rendered as a single space by the browser. Indentation exists purely for developer readability.

<!-- All three render identically: --> <p>Hello World</p> <p>Hello World</p> <p>Hello World</p> <!-- To force whitespace, use: --> <p>Hello&nbsp;&nbsp;&nbsp;World</p> <!-- 3 spaces --> <p>Line one<br>Line two</p> <!-- line break --> <pre>Preserves all spacing</pre>HTML
üí° Industry Standard
Use 2-space indentation for HTML (some teams prefer 4). Every nested element should be indented one level deeper than its parent. This is non-negotiable in professional environments.

2.2 CSS Syntax

Every CSS rule follows the same pattern:

selector { property: value; another-property: another-value; }CSS

Anatomy Breakdown

  • Selector: Targets which HTML element(s) to style
  • Declaration block: The { } containing all rules
  • Declaration: A single property: value; pair
  • Property: What aspect to change (color, font-size, margin...)
  • Value: The setting for that property

Specificity Scoring System

LevelSelector TypeScoreExample
0Universal, combinators0-0-0*, >, +
1Element, pseudo-element0-0-1p, ::before
2Class, attribute, pseudo-class0-1-0.card, [href], :hover

Specificity Calculation Examples

/* 0-0-1 (one element) */ p { color: black; } /* 0-1-0 (one class) */ .intro { color: blue; } /* 0-1-1 (one class + one element) */ p.intro { color: green; } /* 1-0-0 (one ID) */ #hero { color: red; } /* 1-1-1 (one ID + one class + one element) */ #hero .intro p { color: purple; } /* RESULT: <p class="intro"> inside #hero gets color: purple (highest specificity) */CSS
üö´ Avoid

The Box Model

Every single element in CSS is a rectangular box. Understanding the box model is the single most important concept in CSS ‚Äî get this wrong and every layout breaks.

3.1 Box Model Structure

From inside out, every element box consists of four layers:

Visual: Box Model Layers
MARGIN
PADDING
BORDER
CONTENT

box-sizing: The Critical Property

/* DEFAULT: width = content only Total = width + padding + border */ .box-default { box-sizing: content-box; width: 200px; padding: 20px; border: 5px solid black; /* Actual width: 200+40+10 = 250px! */ } /* BETTER: width includes padding + border */ .box-border { box-sizing: border-box; width: 200px; padding: 20px; border: 5px solid black; /* Actual width: exactly 200px ‚úÖ */ } /* ALWAYS apply this universal reset: */ *, *::before, *::after { box-sizing: border-box; }CSS
üîë Employment Essential
Every professional CSS file starts with box-sizing: border-box applied universally. Without it, adding padding or borders to elements with percentage widths will cause layout overflow.

3.2 Margin vs Padding in Practice

Margin Shorthand

/* All four sides */ margin: 10px; /* Vertical | Horizontal */ margin: 10px 20px; /* Top | Horizontal | Bottom */ margin: 10px 20px 30px; /* Top | Right | Bottom | Left (clockwise) */ margin: 10px 20px 30px 40px; /* Center a block element horizontally */ margin: 0 auto;CSS

Margin Collapse ‚Äî The Hidden Trap

Vertical margins collapse: when two vertical margins touch, only the larger one applies.

.box-a { margin-bottom: 30px; } .box-b { margin-top: 20px; } /* Space between them: 30px (NOT 50px!) The larger margin wins. */ /* Margins do NOT collapse when: - Flexbox or Grid parent - Parent has padding or border - Parent has overflow: hidden - Horizontal margins (never collapse) */CSS
Hard

Exercise 3.2A ‚Äî Box Model Calculation Challenge

Calculate the actual rendered width and total horizontal space for each element:

/* Element A */ .element-a { box-sizing: content-box; width: 300px; padding: 15px 25px; border: 3px solid black; margin: 10px 20px; } /* Element B */ .element-b { box-sizing: border-box; width: 300px; padding: 15px 25px; border: 3px solid black; margin: 10px 20px; }CSS
‚úÖ Solution

Element A (content-box):

  • Content width: 300px
  • + Left padding: 25px + Right padding: 25px = 50px
  • + Left border: 3px + Right border: 3px = 6px
  • Rendered box width: 300 + 50 + 6 = 356px
  • + Left margin: 20px + Right margin: 20px = 40px
  • Total horizontal space: 356 + 40 = 396px

Element B (border-box):

  • Total box width: 300px (includes padding + border)
  • Content width: 300 - 50 - 6 = 244px
  • Rendered box width: 300px
  • + Left margin: 20px + Right margin: 20px = 40px
  • Total horizontal space: 300 + 40 = 340px

Section 13

Best Practices

These rules are what make your work maintainable and employable. Most ‚ÄúCSS pain‚Äù is self-inflicted by ignoring these principles.

13.1 The Non-negotiables

  • Prefer semantic HTML and only use <div> when no semantic tag fits.
  • Classes over IDs for styling (IDs for JS hooks/anchor targets only).
  • Avoid fixed widths unless you truly need them. Use max-width, fluid layouts, and min()/max()/clamp().
  • Use a spacing scale (e.g., 4/8/12/16/24/32...) so UIs feel consistent.
  • Use box-sizing: border-box globally.
  • Mobile-first where possible; add breakpoints to enhance layout.
  • Don‚Äôt fight the flow: default document flow + flex/grid beats absolute positioning.

13.2 Modern Responsive Sizing with clamp()

/* clamp(min, preferred, max) */
h1 {
  font-size: clamp(28px, 4vw, 44px);
}

.container {
  width: min(1100px, 92vw);
  margin: 0 auto;
}CSS
