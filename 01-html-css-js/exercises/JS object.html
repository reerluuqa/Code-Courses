What Are Objects?

Objects are collections of related data and functionality, stored as key-value pairs.

üì¶ Objects = Containers for Related Data

In JavaScript, an object is a collection of key-value pairs. Think of it as a real-world thing described in code!

ü™™ Real-World Analogy: An ID Card

Your ID card has: Name ‚Üí "Alex", Age ‚Üí 25, Country ‚Üí "USA". Each piece of information has a label (key) and a value. That's exactly what a JavaScript object does!

Here's your first object ‚Äî a person represented in JavaScript:

// üë§ A person object
const person = {
name: "Alex",
age: 25,
isStudent: true,
greet: function() { ... }
}

üîë Key Terminology

Term What It Means Example
Object A container for related data { name: "Alex" }
Property A key-value pair that holds data name: "Alex"
Key The name/label of a property name
Value The data stored in a property "Alex"
Method A function stored in an object greet: function() {}

üÜö Objects vs Other Data Types

You already know primitive types like strings and numbers. Objects are different ‚Äî they can hold multiple values!

JavaScript
// Primitives hold ONE value let name = "Alex"; let age = 25; let isStudent = true; // Objects hold MANY related values together! üéâ const person = { name: "Alex", age: 25, isStudent: true }; // Much more organized! üì¶
üí° Pro Tip

Objects can hold ANY data type as values: strings, numbers, booleans, arrays, even other objects!

Creating Objects

Learn all the ways to create objects in JavaScript.

üìù Method 1: Object Literal (Most Common!)

The simplest and most common way to create an object. Just use curly braces {}!

JavaScript

üèóÔ∏è Method 2: new Object() Constructor

Less common, but good to know. Creates an empty object, then add properties one by one.

JavaScript

Properties & Access

Learn how to read, add, update, and delete object properties.

üìñ Two Ways to Access Properties

Dot Notation

Clean & simple
‚öîÔ∏è

JavaScript

‚úèÔ∏è Adding & Updating Properties

JavaScript
const car = { brand: "Toyota", year: 2020 }; // ‚ûï Adding new properties car.color = "red"; // dot notation car["fuel"] = "gasoline"; // bracket notation // üîÑ Updating existing properties car.year = 2024; // was 2020, now 2024 car.brand = "Honda"; // was "Toyota", now "Honda" // üóëÔ∏è Deleting properties delete car.fuel; console.log(car.fuel); // undefined // üîç Checking if property exists console.log("brand" in car); // true console.log("fuel" in car); // false (we deleted it)

üèóÔ∏è Nested Object Access

JavaScript
const user = { name: "Sara", address: { street: "456 Oak Ave", city: "Los Angeles", zip: "90001" }, hobbies: ["reading", "gaming", "cooking"] }; // üîó Chain dot notation for nested access console.log(user.address.city); // "Los Angeles" console.log(user.address.zip); // "90001" console.log(user.hobbies[0]); // "reading" console.log(user.hobbies[2]); // "cooking" // ‚ö†Ô∏è Accessing non-existent nested prop console.log(user.phone); // undefined // console.log(user.phone.number); // ‚ùå ERROR! console.log(user.phone?.number); // undefined (safe!)
üí° Optional Chaining (?.)

Use ?. to safely access nested properties. If the left side is undefined or null, it returns undefined instead of crashing!

Object Methods

Methods are functions that live inside objects ‚Äî they give objects the ability to DO things!

ü§ñ Think of Methods Like This...

A phone has properties (brand, color, storage) and methods (call, text, takePhoto). Properties describe WHAT it is. Methods describe what it can DO!

üìù Defining Methods

JavaScript
const dog = { name: "Rex", breed: "Labrador", energy: 100, // Method 1: Function expression (old way) bark: function() { return "Woof! Woof! üêï"; }, // Method 2: Shorthand (modern & preferred ‚úÖ) sit() { return "*sits down* üêï"; }, // Method with parameters fetch(item) { return `${this.name} fetches the ${item}! üéæ`; } }; // üìû Calling methods console.log(dog.bark()); // "Woof! Woof! üêï" console.log(dog.sit()); // "*sits down* üêï" console.log(dog.fetch("ball")); // "Rex fetches the ball! üéæ"
‚ö†Ô∏è Common Mistake

Don't forget the parentheses () when calling methods! dog.bark gives you the function itself. dog.bark() actually calls it.

JavaScript

The this Keyword

One of the trickiest but most important concepts ‚Äî let's demystify it!

ü™û Think of "this" as a Mirror

When you say "I am hungry," the word "I" refers to you ‚Äî the person speaking. In JavaScript, this works the same way. Inside an object's method, this refers to the object that called the method.

üëÜ this In Object Methods

JavaScript
const user = { firstName: "John", lastName: "Doe", age: 30, // 'this' refers to the 'user' object getFullName() { return this.firstName + " " + this.lastName; // ^^^^ ^^^^ // "John" "Doe" }, introduce() { return `Hi! I'm ${this.getFullName()}, age ${this.age}`; } }; console.log(user.getFullName()); // "John Doe" console.log(user.introduce()); // "Hi! I'm John Doe, age 30"

Object Destructuring

A beautiful way to unpack values from objects into separate variables!

üéÅ Think of it as Unwrapping a Gift Box

Imagine you have a gift box (object) with items inside. Destructuring lets you reach in and grab exactly what you want, giving each item its own name. Instead of carrying the whole box around, you just take what you need!

üì§ Basic Destructuring

JavaScript
const person = { name: "Alex", age: 25, city: "New York" }; // üò¥ The OLD way (repetitive!) const name1 = person.name; const age1 = person.age; const city1 = person.city; // üéâ The NEW way (destructuring!) const { name, age, city } = person; console.log(name); // "Alex" console.log(age); // 25 console.log(city); // "New York"

üé® Renaming Variables

You can rename the variables while destructuring!

JavaScript
const user = { name: "Emma", age: 22 }; // Rename: { originalKey: newName } const { name: userName, age: userAge } = user; console.log(userName); // "Emma" console.log(userAge); // 22 // console.log(name); // ‚ùå Error! 'name' doesn't exist

üõ°Ô∏è Default Values

JavaScript
const settings = { theme: "dark", fontSize: 16 }; // Default values for missing properties const { theme, fontSize, language = "English", // default! notifications = true // default! } = settings; console.log(theme); // "dark" (from object) console.log(language); // "English" (default used) console.log(notifications); // true (default used)

üèóÔ∏è Nested Destructuring

JavaScript
const employee = { name: "Sara", role: "Developer", address: { city: "Austin", state: "TX", zip: "78701" } }; // Nested destructuring const { name, role, address: { city, state } } = employee; console.log(name); // "Sara" console.log(city); // "Austin" console.log(state); // "TX"

‚ö° Destructuring in Function Parameters

Super common in real code! Instead of passing a whole object, destructure right in the parameters.

JavaScript
// üò¥ Without destructuring function greetOld(user) { return `Hello ${user.name}, age ${user.age}!`; } // üéâ With destructuring in parameters! function greet({ name, age }) { return `Hello ${name}, age ${age}!`; } const user = { name: "Alex", age: 25, city: "NYC" }; console.log(greet(user)); // "Hello Alex, age 25!" // With defaults in parameters function createUser({ name, role = "user", active = true }) { return `${name} is a ${role} (active: ${active})`; } console.log(createUser({ name: "Bob" })); // "Bob is a user (active: true)"

Object.keys / values / entries

Built-in methods to explore and iterate over objects!

üîë Object.keys() ‚Äî Get All Keys

Returns an array of all the property names (keys).

JavaScript
const fruit = { name: "Apple", color: "red", calories: 95, tasty: true }; const keys = Object.keys(fruit); console.log(keys); // ["name", "color", "calories", "tasty"] // Great for counting properties! console.log(`Fruit has ${keys.length} properties`); // "Fruit has 4 properties" // Loop through keys keys.forEach(key => { console.log(`${key}: ${fruit[key]}`); });

üíé Object.values() ‚Äî Get All Values

Returns an array of all property values.

JavaScript
const scores = { math: 95, science: 88, english: 92, history: 87 }; const values = Object.values(scores); console.log(values); // [95, 88, 92, 87] // Calculate average! const total = values.reduce((sum, val) => sum + val, 0); const average = total / values.length; console.log(`Average: ${average}`); // Average: 90.5 // Find highest score const highest = Math.max(...values); console.log(`Highest: ${highest}`); // Highest: 95

üìã Object.entries() ‚Äî Get Key-Value Pairs

Returns an array of [key, value] pairs. Perfect for looping!

JavaScript
const prices = { coffee: 4.99, tea: 3.49, juice: 5.99, water: 1.99 }; const entries = Object.entries(prices); console.log(entries); // [["coffee", 4.99], ["tea", 3.49], ["juice", 5.99], ["water", 1.99]] // Loop with destructuring! üéâ for (const [drink, price] of entries) { console.log(`‚òï ${drink}: $${price}`); } // Find items under $4 const affordable = entries .filter(([_, price]) => price < 4) .map(([drink]) => drink); console.log(`Under $4: ${affordable}`); // Under $4: tea,water

üÜö Quick Comparison

Method Returns Example Output
Object.keys(obj) Array of keys ["name", "age"]
Object.values(obj) Array of values ["Alex", 25]
Object.entries(obj) Array of [key, value] [["name","Alex"],["age",25]]

üí™ Exercise 4: Object.keys/values Challenge

Medium

Given the scores object, use Object.values() to calculate the average score and Object.keys() to count subjects.

Use .length to count keys. Use .reduce((sum, val) => sum + val, 0) to sum values, then divide by length.

üí™ Exercise 5: Bank Account Object

Hard

Create a bankAccount object with: owner, balance (start at 1000), deposit(amount) method, withdraw(amount) method (check if enough funds!), and getStatement() method.

Use this.balance to access and modify balance. In withdraw, check if (amount > this.balance) first. Return messages from each method.
const bankAccount = {
    owner: "Alex",
    balance: 1000,
    
    deposit(amount) {
        this.balance += amount;
        return `Deposited $${amount}. Balance: $${this.balance}`;
    },
    
    withdraw(amount) {
        if (amount > this.balance) {
            return `Insufficient funds! Balance: $${this.balance}`;
        }
        this.balance -= amount;
        return `Withdrew $${amount}. Balance: $${this.balance}`;
    },
    
    getStatement() {
        return `${this.owner}'s account: $${this.balance}`;
    }
};

console.log(bankAccount.deposit(500));
console.log(bankAccount.withdraw(200));
console.log(bankAccount.getStatement());

üí™ Exercise 6: Object.entries Transform

Hard

Given an object of temperatures in Celsius, use Object.entries() to create a NEW object with temperatures converted to Fahrenheit (F = C √ó 9/5 + 32).

Use Object.entries(celsius).map(([day, temp]) => [day, temp * 9/5 + 32]) then wrap with Object.fromEntries().
const fahrenheit = Object.fromEntries(
    Object.entries(celsius).map(([day, c]) => [day, c * 9/5 + 32])
);

console.log(fahrenheit);
// { monday: 68, tuesday: 77, wednesday: 64.4, thursday: 71.6, friday: 86 }

Common Pitfalls & Mistakes

Learn from the most common mistakes JavaScript beginners make with objects!

‚ùå Pitfall 1: Forgetting parentheses when calling methods

‚ùå WRONG
console.log(dog.bark); // logs the function itself! // ∆í () { return "Woof!"; }
VS
‚úÖ CORRECT
console.log(dog.bark()); // "Woof!" ‚úÖ // () calls the function

‚ùå Pitfall 2: Using arrow functions for methods with this

‚ùå WRONG
const obj = { name: "Alex", greet: () => { return this.name; // undefined! ‚ùå } };
VS
‚úÖ CORRECT
const obj = { name: "Alex", greet() { return this.name; // "Alex" ‚úÖ } };

‚ùå Pitfall 3: Thinking const prevents property changes

JavaScript
const user = { name: "Alex", age: 25 }; // ‚úÖ This WORKS! You can change properties user.age = 26; // OK! user.email = "a@b.com"; // OK! // ‚ùå This FAILS! Can't reassign the variable // user = { name: "Bob" }; // TypeError! // üí° const prevents REASSIGNMENT, not MUTATION

‚ùå Pitfall 4: Accessing nested properties without checking

‚ùå CRASHES
const user = { name: "Alex" }; console.log(user.address.city); // TypeError: Cannot read // property of undefined üí•
VS
‚úÖ SAFE
const user = { name: "Alex" }; console.log(user.address?.city); // undefined ‚úÖ // Optional chaining saves us!

‚ùå Pitfall 5: Comparing objects with ===

JavaScript
const a = { name: "Alex" }; const b = { name: "Alex" }; const c = a; console.log(a === b); // false! üò± Different objects in memory console.log(a === c); // true! Same reference // To compare CONTENTS, use JSON.stringify: console.log(JSON.stringify(a) === JSON.stringify(b)); // true ‚úÖ

‚ùå Pitfall 6: Destructuring with wrong variable names

‚ùå WRONG
const obj = { name: "Alex" }; const { username } = obj; console.log(username); // undefined ‚ùå // 'username' doesn't exist!
VS
‚úÖ CORRECT
const obj = { name: "Alex" }; const { name: username } = obj; console.log(username); // "Alex" ‚úÖ // Rename with : syntax
