Chapter 1

What Are Objects?

Think of an object as a container that holds related data and functionality together. It's like a backpack with labeled pockets - each pocket (property) has a name and contains something valuable.

name "Alex"
age 28
isDeveloper true
greet function() {...}

Hover over properties to see them highlight

person.js 
const person = {
  name: "Alex",      // String property
  age: 28,            // Number property
  isDeveloper: true, // Boolean property
  greet: function() { // Method
    return `Hi, I'm ${this}.name`;
  }
};

Properties

Data stored in an object. Like labeled boxes holding values: strings, numbers, booleans, or even other objects.

Methods

Functions inside objects. They can access other properties using the this keyword.

Key-Value Pairs

Every property has a key (name) and a value. Keys are always strings (or Symbols), values can be anything.

Chapter 2

Creating Objects

There are multiple ways to create objects in JavaScript. Let's explore each one.

Object Literal (Most Common)

The simplest and most popular way. Just use curly braces with your properties inside.

Best for: Single objects, configuration, quick data structures
// Object literal syntax
const user = {
  name: "Sarah",
  email: "sarah@example.com",
  age: 25
};

// Empty object, add properties later
const settings = {};
settings.theme = "dark";
settings.notifications = true;

Pro tip: Use shorthand when variable name matches property name:

const name = "John"; const user = { name }; // Same as { name: name }

Using new Object()

Less common, but good to know. Creates an empty object that you populate afterwards.

Rarely used, but understanding it helps
const car = new Object();
car.brand = "Tesla";
car.model = "Model 3";
car.year = 2024;

// This is equivalent to:
const car = {
  brand: "Tesla",
  model: "Model 3",
  year: 2024
};

Constructor Function

Create multiple objects of the same type. Like a blueprint for objects.

Best for: Creating multiple similar objects
// Constructor function (capitalize first letter)
function Dog(name, breed, age) {
  this.name = name;
  this.breed = breed;
  this.age = age;
  this.bark = function() {
    return `${this} says Woof!`;
  };
}

// Create instances with 'new'
const buddy = new Dog("Buddy", "Golden", 3);
const max = new Dog("Max", "Bulldog", 5);

console.log(buddy.bark()); // "Buddy says Woof!"

ES6 Class Syntax

Modern, cleaner syntax for creating object blueprints. Syntactic sugar over constructors.

Modern approach (ES6+)
class Dog {
  constructor(name, breed, age) {
    this.name = name;
    this.breed = breed;
    this.age = age;
  }

  bark() {
    return `${this}.name says Woof!`;
  }

  getInfo() {
    return `${this}.name is a ${this}.age} year old ${this}.breed`;
  }
}

const luna = new Dog("Luna", "Husky", 2);
Chapter 3

Accessing Properties

Two ways to access object properties: dot notation and bracket notation.

Dot Notation

Clean and simple. Use when you know the property name beforehand.

const user = {
  name: "Emma",
  age: 30
};

console.log(user.name);  // "Emma"
console.log(user.age);   // 30

// Modify property
user.age = 31;

// Add new property
user.email = "emma@test.com";

Use when: Property name is valid identifier (no spaces, doesn't start with number)

Bracket Notation

More flexible. Use with variables or special property names.

const user = {
  name: "Emma",
  "full name": "Emma Watson",
  "1stPlace": true
};

// Access with string
console.log(user["name"]);       // "Emma"
console.log(user["full name"]); // "Emma Watson"

// Access with variable
const key = "name";
console.log(user[key]); // "Emma"

Use when: Property has spaces, starts with number, or is in a variable

Chapter 4

Methods & The this Keyword

Methods are functions that live inside objects. The this keyword refers to the object itself.

Understanding 'this'
const wizard = {
  name: "Gandalf",
  mana: 100,
  
  // Method using this
  castSpell(spellName) {
    console.log(`${this}.name casts ${spellName}!`);
    this.mana -= 10;
  },
  
  getStatus() {
    return `${this}.name has ${this}.mana mana`;
  }
};

wizard.castSpell("Fireball");
console.log(wizard.getStatus());

Arrow Functions & this

Arrow functions don't have their own this. They inherit it from the surrounding scope.

// DON'T use arrow functions for methods that need 'this'
const bad = {
  name: "Test",
  greet: () => {
    console.log(this.name); // undefined! Arrow function has no 'this'
  }
};

// DO use regular function syntax
const good = {
  name: "Test",
  greet() {  // Method shorthand (recommended)
    console.log(this.name); // "Test"
  }
};
Chapter 5

Object Destructuring

Extract properties from objects into variables with a elegant, clean syntax.

Basic Syntax

const user = {
  name: "Alice",
  age: 28,
  city: "Berlin"
};

// Old way (boring)
const name = user.name;
const age = user.age;

// Destructuring (cool)
const { name, age, city } = user;

console.log(name); // "Alice"
console.log(age);  // 28

Advanced Techniques

// Rename variables
const { name: userName, age: userAge } = user;
console.log(userName); // "Alice"

// Default values
const { name, country = "Unknown" } = user;
console.log(country); // "Unknown"

// Rest operator (collect remaining)
const { name, ...rest } = user;
console.log(rest); // { age: 28, city: "Berlin" }

// Nested destructuring
const { address: { street } } = complexUser;

Chapter 6

Object.keys / values / entries

Convert objects into arrays for easy iteration and transformation.

Object.keys()

Returns an array of property names

const obj = { a: 1, b: 2 };
Object.keys(obj);
// ["a", "b"]

Object.values()

Returns an array of property values

const obj = { a: 1, b: 2 };
Object.values(obj);
// [1, 2]

Object.entries()

Returns array of [key, value] pairs

const obj = { a: 1, b: 2 };
Object.entries(obj);
// [["a", 1], ["b", 2]]

Practical Examples

Real-world usage patterns
const scores = {
  Alice: 95,
  Bob: 82,
  Charlie: 78,
  Diana: 91
};

// Get all student names
const students = Object.keys(scores);
// ["Alice", "Bob", "Charlie", "Diana"]

// Calculate average score
const avg = Object.values(scores).reduce((a, b) => a + b) / Object.values(scores).length;
// 86.5

// Iterate with entries
Object.entries(scores).forEach(([name, score]) => {
  console.log(`${name}: ${score}`);
});

// Transform to new object
const grades = Object.fromEntries(
  Object.entries(scores).map(([name, score]) => [
    name,
    score >= 90 ? "A" : score >= 80 ? "B" : "C"
  ])
);
// { Alice: "A", Bob: "B", Charlie: "C", Diana: "A" }

Watch Out

Common Pitfalls

Avoid these frequent mistakes when working with objects.

Mutating objects you don't own

Never modify objects that you didn't create (like builtâ€‘in prototypes).

// BAD: Don't extend Array.prototype
Array.prototype.first = function() { return this[0]; };

// GOOD: Create your own utility
function getFirst(arr) { return arr[0]; }

Comparing objects by value

Two objects with the same contents are not equal unless they reference the same object.

const a = { name: "John" };
const b = { name: "John" };
console.log(a === b); // false

// Compare by converting to JSON (simple case)
console.log(JSON.stringify(a) === JSON.stringify(b)); // true

Forgetting 'new' with constructors

Calling a constructor without new pollutes the global object.

function Person(name) {
  this.name = name;
}

// BAD - 'this' refers to window/global
const p = Person("Alex");
console.log(name); // "Alex" - leaked global!

// GOOD - use 'new'
const p2 = new Person("Alex");

Using reserved keywords as property names

Avoid names like class, delete, new (works, but confusing).

// Works but can cause issues in strict mode
const obj = {
  class: "math",
  delete: true
};

// Better to use valid identifiers
const obj2 = {
  className: "math",
  canDelete: true
};
