‚ö†Ô∏è Common Pitfalls

üö´ HTTPS Required

Geolocation only works on secure origins (HTTPS or localhost).

Solution: Always test on HTTPS or use localhost during development. Never rely on it working on HTTP.
üö´ Permission Denial Handling

Users can deny permission, and you must handle this gracefully.

Solution: Check error.code: 1=Denied, 2=Unavailable, 3=Timeout. Provide fallback UI.
üö´ Battery Drain

watchPosition with high accuracy drains battery fast.

Solution: Use maximumAge to cache results. Stop watching when not needed. Consider enableHighAccuracy: false for approximate location.

‚ö†Ô∏è Critical Pitfalls

üö´ No DOM Access

Workers cannot access window, document, or any DOM elements.

Solution: Pass data to main thread to update UI. Use self instead of window in workers.
üö´ Memory Leaks

Creating workers without terminating them consumes memory.

Solution: Always call worker.terminate() when done, or use worker.onerror to handle crashes.
üö´ Same-Origin Policy

Worker scripts must be from the same origin (or use CORS).

Solution: Use Blob URLs for inline workers if needed, or ensure proper CORS headers.
üö´ Serialization Cost

Data passed to workers is cloned (structured clone), not shared.

Solution: Use Transferable Objects (like ArrayBuffer) for large data to avoid copying: postMessage(data, [buffer])

// sw.js - Service Worker File
const CACHE_NAME = 'v1';
const urlsToCache = ['/', '/styles.css', '/app.js'];
// Install event
self.addEventListener('install', event => {
event.waitUntil(
caches.open(CACHE_NAME)
.then(cache => cache.addAll(urlsToCache))
);
});
// Fetch interceptor
self.addEventListener('fetch', event => {
event.respondWith(
caches.match(event.request)
.then(response => {
// Cache hit - return response
if (response) return response;
return fetch(event.request); // Network
})
);
});

üéì Caching Strategies

Cache First

Fastest. Best for: Static assets, App Shell

return cache || fetch()

Network First

Fresh data. Best for: API calls, Dynamic content

return fetch() || cache

Stale While Revalidate

Fast + Update. Best for: Content that can be slightly old

‚ö†Ô∏è Deadly Pitfalls

üö´ Scope Issues

SW only controls pages in its directory and subdirectories.

Solution: Place sw.js at root (/) to control entire site. Check navigator.serviceWorker.controller
üö´ Update Cycle

New SW waits until all tabs are closed before activating.

Solution: Use self.skipWaiting() in install event and clients.claim() in activate event.
üö´ Cache Bloat

Old caches accumulate and exceed storage quotas.

Solution: Delete old caches in activate event: caches.keys().then(names => names.filter(n => n !== CURRENT).map(n => caches.delete(n)))
Implement a "Cache, then Network" strategy for a news feed. Show cached content immediately, then update when network responds.

‚ö†Ô∏è Production Pitfalls

üö´ No Auto-Reconnect

WebSockets don't reconnect automatically when connection drops.

Solution: Implement exponential backoff reconnection logic. Use libraries like Socket.io or reconnecting-websocket.
üö´ Memory Leaks

Not removing event listeners causes memory leaks on reconnect.

Solution: Always call removeEventListener or use { once: true } option. Clean up on close.
üö´ Blocking the UI

Processing large messages on message event blocks the main thread.

Solution: Delegate heavy processing to Web Workers. Or use requestIdleCallback for non-critical updates.
üö´ Security (WS vs WSS)

Using ws:// (unencrypted) on HTTPS pages is blocked.

Solution: Always use wss:// (WebSocket Secure) in production. Implement proper origin checking on server.

‚ö†Ô∏è UX Pitfalls

üö´ Asking on Page Load

Requesting permission immediately annoys users and leads to denials.

Solution: Request permission only after user interaction (clicking "Enable Notifications" button).
üö´ No Service Worker = No Push

Push notifications (server-sent) require a Service Worker.

Solution: Local notifications work without SW, but Push API requires Service Worker registration and server keys (VAPID).
üö´ Mobile Quirks

iOS only supports Push in PWAs added to home screen. Android kills background apps.

Solution: Check platform. Implement push for Android, in-app notifications for iOS Safari (until full support arrives).
