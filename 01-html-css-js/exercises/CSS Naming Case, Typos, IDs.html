Learning notes (newbie-friendly, but accurate) rules that prevent 80% of CSS bugs
  1. Selectors must match exactly.
    Case drift is a silent failure mode: class="Middle" will not match .middle. The browser won’t warn you; it simply won’t apply the rules.
  2. Use a naming convention that eliminates case mistakes.
    Prefer lowercase + hyphens (kebab-case): header-nav, nav-center, header-actions. Avoid MainHeader / Middle unless you have a strict team standard and linting.
  3. Don’t “fix” a naming bug with extra CSS.
    If spacing “doesn’t work,” check: (1) class name typo, (2) case, (3) selector match in DevTools, (4) specificity only after those.
  4. IDs are not for styling (most of the time).
    IDs have higher specificity and can cause unexpected overrides. Use IDs for document anchors and JS hooks; use classes for styling and layout.
  5. IDs must be unique.
    Duplicate IDs break assumptions: anchor navigation goes to the first match; JS selection becomes unpredictable. Treat this as a correctness defect, not a cosmetic issue.
Exercise: In the first demo, change class="Middle" to class="middle" and watch the spacing change immediately. Then try renaming .middle in CSS and see how fast things break.
DevTools workflow: Inspect the element → look at the “Styles” pane. If your selector is missing entirely, it’s usually a naming mismatch. If it’s crossed out, it’s usually specificity/order.
