The Almighty Reduce

Transform any array into any value: objects, arrays, numbers, or promises

reduce() is the most powerful array method. It takes an accumulator and current value, allowing you to build anything from simple sums to complex data structures.

Essential Reduce Patterns

1. Group By (The Interview Favorite)

const people = [ { name: 'Alice', age: 21, group: 'A' }, { name: 'Bob', age: 25, group: 'B' }, { name: 'Charlie', age: 21, group: 'A' } ]; const groupedByAge = people.reduce((acc, person) => { const key = person.age; if (!acc[key]) acc[key] = []; acc[key].push(person); return acc; }, {}); // Result: { 21: [Alice, Charlie], 25: [Bob] }

2. Function Pipeline (Compose)

const pipe = (...fns) => (x) => fns.reduce((v, f) => f(v), x); const add5 = x => x + 5; const multiply2 = x => x * 2; const toString = x => `Result: ${x}`; const transform = pipe(add5, multiply2, toString); transform(5); // "Result: 20"

3. Sequential Async Operations

// Run promises in sequence, not parallel const urls = ['/api/1', '/api/2', '/api/3']; const results = await urls.reduce(async (acc, url) => { const prev = await acc; const response = await fetch(url); const data = await response.json(); return [...prev, data]; }, Promise.resolve([]));

‚ò†Ô∏è Common Pitfalls

The Empty Array Trap

Without initial value on empty array = CRASH

[].reduce((a, b) => a + b); // TypeError: Reduce of empty array with no initial value
Fix: Always provide initial value
[].reduce((a, b) => a + b, 0)
Mutation Mayhem

Mutating the accumulator reference

acc.push(item); // Mutates! return acc;
Fix: Return new reference
return [...acc, item];
Or use .concat() for objects: {...acc, [key]: val}
Forgotten Return

Implicit undefined return

array.reduce((acc, curr) => { acc.total += curr; // No return statement! }, {total: 0})
Fix: Always return acc
Add return acc; at end

üöÄ FlatMap: The Power Combo

flatMap() = map() + flat(1) in one efficient pass. Perfect for: filtering + mapping, adding/removing items during map, or handling optional values.

Classic Use Case: Sentence Tokenization

const sentences = [ "Hello world", "FlatMap is powerful" ]; // Split into words and flatten in one go const words = sentences.flatMap(s => s.split(' ')); // ['Hello', 'world', 'FlatMap', 'is', 'powerful'] // vs inefficient: // sentences.map(s => s.split(' ')).flat();
Performance Win: FlatMap does it in O(n) vs O(2n) for map+flat. No intermediate array creation!

Clever Pattern: Filter + Map

const numbers = [1, 2, 3, 4, 5]; // Keep only evens, then double them const doubledEvens = numbers.flatMap(n => { if (n % 2 === 0) return [n * 2]; // Keep as array return []; // Remove by returning empty }); // [4, 8] // vs: numbers.filter(n => n % 2 === 0).map(n => n * 2)
Note: Only flats 1 level deep. For deeper nesting, use map().flat(depth).

// ‚ùå Hard to debug (where did it go wrong?) const result = users .filter(u => u.active) .map(u => u.scores.reduce((a,b)=>a+b)/u.scores.length) .reduce((a,b)=>a+b,0) / users.length; // ‚úÖ Debuggable pipeline with named steps const activeUsers = users.filter(u => u.active); const averages = activeUsers.map(u => { const sum = u.scores.reduce((a,b) => a+b, 0); return sum / u.scores.length; }); const totalAvg = averages.reduce((a,b) => a+b, 0) / averages.length;

Performance Impact

Multiple Iterations O(3n) - 3 loops
Single Reduce (fusion) O(n) - 1 loop
Pro Tip: For massive datasets (>100k items), consider a single reduce or for-loop to avoid iteration overhead. For <10k items, prefer readability!

Advanced Patterns

Lazy Evaluation with Generators

function* lazyChain(arr) { for (const item of arr) { if (item.active) { // filter yield item.value * 2; // map } } } // Processes one item at a time, memory efficient const result = [...lazyChain(largeArray)];

Transducers (Function Composition)

const map = f => next => (acc, x) => next(acc, f(x)); const filter = p => next => (acc, x) => p(x) ? next(acc, x) : acc; const pipeline = filter(x => x > 2)(map(x => x * 2)(concat)); [1,2,3,4].reduce(pipeline, []); // [6, 8] - single pass!
