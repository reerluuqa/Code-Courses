üí° Key Insight

async/await lets you write asynchronous code that looks and behaves like synchronous code, making it easier to read and maintain while still allowing other operations to run during wait times.

Async/Await Syntax Core

Master the fundamental building blocks of modern asynchronous JavaScript.

1. The Async Function

Adding async before a function automatically makes it return a Promise.

JavaScript 
// Regular function returns value
function regular() {
    return "Hello";
}

// Async function ALWAYS returns a Promise
async function asyncFunc() {
    return "Hello"; // Returns Promise<string>
}

// These are equivalent:
async function explicit() {
    return Promise.resolve("Hello");
}

2. The Await Keyword

await pauses execution until the Promise settles, but only inside async functions.

JavaScript 
async function getUserData() {
    console.log("Fetching...");
    
    // Execution PAUSES here, but doesn't block the main thread
    const response = await fetch('/api/user');
    const data = await response.json();
    
    console.log("Done!");
    return data;
}
Quick Check: What does this function return?

    return 42;
}

Error Handling Dojo üõ°Ô∏è

Async code fails differently. Learn to catch errors gracefully.

The Try-Catch-Finally Pattern

Always wrap await calls in try-catch blocks. Unlike synchronous code, uncaught async errors become unhandled promise rejections.

Robust Pattern 
async function fetchUserData(userId) {
    try {
        console.log("Fetching user...");
        
        const response = await fetch(`/api/users/${userId}`);
        
        // Check for HTTP errors (404, 500, etc)
        if (!response.ok) {
            throw new Error(`HTTP error! status: ${response.status}`);
        }
        
        const data = await response.json();
        return data;
        
    } catch (error) {
        console.error("Failed to fetch user:", error.message);
        // Return default value or re-throw
        return null;
    } finally {
        console.log("Cleanup: Hide loading spinner");
    }
}
Dangerous Pattern 
// ‚ùå BAD: Missing try-catch
async function riskyFunction() {
    // If fetch fails, this becomes an unhandled rejection!
    const res = await fetch('/api/data');
    return await res.json();
}

// ‚ùå BAD: Catching in the wrong place
try {
    riskyFunction(); // Forgot await!
} catch (e) {
    // This won't catch errors from the async function
    console.log("This won't run");
}

Parallel Error Handling with Promise.all

async function fetchMultiple() {
    try {
        // If one fails, all fail (fail fast)
        const [users, posts] = await Promise.all([
            fetch('/users'),
            fetch('/posts')
        ]);
    } catch (error) {
        // Catches the first rejection
    }
}

The Pitfall Minefield ‚ö†Ô∏è

Learn from others' mistakes. These bugs cost developers hours of debugging.

Pitfall #1: The Forgotten Await

async function getData() {
    const data = fetchData(); // ‚ùå Missing await!
    console.log(data); // Promise {<pending>}
}

You get a Promise object instead of the resolved value.

‚úÖ The Fix

const data = await fetchData();

Always await promises if you need the value.

Pitfall #2: Array Methods

// ‚ùå forEach doesn't wait!
urls.forEach(async (url) => {
    await fetch(url); // Fire and forget
});
console.log("Done"); // Logs before fetches complete

‚úÖ Use for...of instead

for (const url of urls) {
    await fetch(url); // Waits for each
}
// Or Promise.all for parallel:
await Promise.all(urls.map(url => fetch(url)));

Pitfall #3: Sequential vs Parallel

// ‚ùå Slow: Runs one after another
const a = await fetch('/a');
const b = await fetch('/b');
const c = await fetch('/c');
// Total time: 3 seconds

‚úÖ Run in Parallel

const [a, b, c] = await Promise.all([
    fetch('/a'),
    fetch('/b'),
    fetch('/c')
]);
// Total time: 1 second

Pitfall #4: The Async IIFE

// Top-level await not available in older envs
const data = await fetchData(); // ‚ùå SyntaxError

‚úÖ Wrap in Async IIFE

(async () => {
    const data = await fetchData();
    console.log(data);
})();

Async Patterns Workshop üé®

Professional patterns for common scenarios.

Pattern 1: Retry with Exponential Backoff

Production Pattern 
async function fetchWithRetry(url, maxRetries = 3) {
    for (let i = 0; i < maxRetries; i++) {
        try {
            const response = await fetch(url);
            if (!response.ok) throw new Error('Failed');
            return await response.json();
        } catch (error) {
            if (i === maxRetries - 1) throw error;
            
            const delay = Math.pow(2, i) * 1000; // 1s, 2s, 4s
            console.log(`Retrying in ${delay}ms...`);
            await new Promise(resolve => setTimeout(resolve, delay));
        }
    }
}

Pattern 2: Parallel with Concurrency Limit

Don't overwhelm the server. Process 3 items at a time.

async function batchProcess(items, batchSize = 3) {
    const results = [];
    
    for (let i = 0; i < items.length; i += batchSize) {
        const batch = items.slice(i, i + batchSize);
        const batchResults = await Promise.all(
            batch.map(item => processItem(item))
        );
        results.push(...batchResults);
    }
    
    return results;
}

Pattern 3: Timeout Wrapper

function withTimeout(promise, ms) {
    const timeout = new Promise((_, reject) => 
        setTimeout(() => reject(new Error('Timeout!')), ms)
    );
    return Promise.race([promise, timeout]);
}

// Usage
try {
    const data = await withTimeout(fetchData(), 5000);
} catch (e) {
    // Handle timeout
}
