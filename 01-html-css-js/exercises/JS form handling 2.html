The preventDefault() Power

Stopping the browser's default behavior

Without preventDefault(), your form will reload the page and lose all data. This is the #1 mistake in junior dev interviews.

üíÄ The Problem (Don't do this)

‚òùÔ∏è Click this to see the page flash/reload (data lost!)

‚úÖ The Solution

‚úÖ Form handled! Data captured, no reload.

Page already reloading! }); // ‚úÖ CORRECT: Intercept the event immediately form.addEventListener('submit', (event) => { event.preventDefault(); // STOP! Hammer time. // Now safely handle the data const data = new FormData(event.target); // ... send to API });

Validation Patterns

// Strategy: Validate on input, show error on blur const validators = { username: (val) => /^[a-zA-Z0-9]{3,15}$/.test(val), email: (val) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(val), password: (val) => val.length >= 8 && /\d/.test(val) && /[!@#$%^&*]/.test(val) }; // Real-time feedback without being annoying input.addEventListener('input', (e) => { const isValid = validators.username(e.target.value); if (isValid) { e.target.classList.add('success'); e.target.classList.remove('error'); } });

‚ö†Ô∏è Security Warning

Never rely solely on client-side validation. Always validate again on the server. JavaScript validation is for UX, not security.

üí° Pro Tip: Debouncing

For expensive validations (like checking username availability), use debouncing to avoid API spam.

The FormData API

Modern data extraction (No more document.getElementById!)

The FormData API is the professional way to handle form data. It automatically collects all fields, handles files, and formats data for HTTP requests.

üß™ FormData Playground

Extracted Data:

üÜö Old vs New Way

‚ùå The Old Painful Way

const name = document.getElementById('name').value; const email = document.getElementById('email').value; const phone = document.getElementById('phone').value; // Repeat for 20 fields... üò≠

‚úÖ The FormData Way

form.addEventListener('submit', (e) => { e.preventDefault(); const formData = new FormData(e.target); // Convert to regular object const data = Object.fromEntries(formData); // Handle arrays (checkboxes) const skills = formData.getAll('skills'); console.log(data); });

üöÄ Sending to API

// JSON payload fetch('/api/submit', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(data) }); // Or send as FormData (includes files) fetch('/api/submit', { method: 'POST', body: formData // No Content-Type header needed! });

Pitfalls & Job Interview Traps

Mistakes that cost you the job

‚ö†Ô∏è Pitfall #1: Forgetting name attributes

FormData can't extract values from inputs without name attributes. Always add name="fieldName" to every input.

// ‚ùå Missing name - FormData can't see this! <input type="text" id="username"> // ‚úÖ Correct <input type="text" id="username" name="username">

‚ö†Ô∏è Pitfall #2: Not handling Enter key in single-field forms

Pressing Enter in a single input submits the form even without a submit button. Always handle this!

‚ö†Ô∏è Pitfall #3: Memory leaks from event listeners

Attaching listeners inside loops or without cleanup in SPAs causes memory leaks.

‚úÖ Solution: Use event delegation

// Instead of attaching to every input... form.addEventListener('input', (e) => { if (e.target.matches('[data-validate]')) { validateField(e.target); } });

‚ö†Ô∏è Pitfall #4: Race conditions in async validation

If validation involves an API call, older responses might arrive after newer ones, showing stale errors.

‚úÖ Solution: AbortController or timestamp comparison

üìã Quick Reference Cheat Sheet

Event Order

  • 1. input (every keystroke)
  • 2. change (on blur if modified)
  • 3. blur (field loses focus)
  • 4. submit (form submission)

Essential Methods

  • e.preventDefault() - Stop default action
  • formData.get(name) - Get single value
  • formData.getAll(name) - Get array
  • Object.fromEntries(formData) - Convert to object

Validation Regex

  • Email: ^[^\s@]+@[^\s@]+\.[^\s@]+$
  • Phone: ^\d{10}$
  • Alphanumeric: ^[a-zA-Z0-9]+$
