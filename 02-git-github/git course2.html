1. Introduction: What is Git?

Version Control Systems (VCS)

Git is a distributed version control system. It helps developers track changes in their codebase, collaborate with others, and manage multiple versions of a project. Unlike centralized systems, Git allows every developer to have a full copy of the repository locally.

Key Features:

  • Decentralized (No need for a central server locally).
  • Branching and Merging.
  • Speed and Data Integrity (SHA-1 hashing).
  • Used by 94% of developers (Stack Overflow Survey).

Note: Git is the tool (engine); GitHub is the hosting platform.

2. Installation & Configuration

Before using Git, you must configure your identity. These details appear in every commit you make.

# Set your name
git config --global user.name "Brad Traversey"

# Set your email
git config --global user.email "test@test.com"

# Check your config
git config --global user.name

# Optional: Change default branch name to 'main' (to match GitHub)
git config --global init.defaultBranch main
3. Starting a Repository

To start tracking a project, you need to initialize a repository. This creates a hidden .git folder.

# Initialize a new repository
git init

# Remove a repo (Destructive!)
rm -rf .git
4. Staging Files (git add)

Git has a Staging Area. You must add files to the staging area before committing them to the repository.

# Check status of files
git status

# Add a specific file
git add index.html

# Add all files in current directory
git add .
5. Committing Changes

Committing takes the files from the staging area and saves a snapshot to the local repository.

# Commit with a message
git commit -m "Initial commit"

# Shortcut: Add all tracked files AND commit (skips git add)
git commit -a -m "Added task six"

# View commit history
git log
6. Connecting to GitHub

To push your code to GitHub, create a repository on GitHub first, then link it to your local machine.

# Link local repo to remote URL (alias 'origin')
git remote add origin https://github.com/username/repo-name.git

# Rename branch to main (if not already)
git branch -M main
7. Pushing Code

Sending your commits to the remote server.

# First push (sets upstream tracking)
git push -u origin main

# Subsequent pushes
git push
8. Pulling Changes

If changes exist on GitHub (e.g., from another developer or a README created online), pull them down to your local machine.

# Pull latest changes
git pull origin main
9. Cloning Repositories

To download an entire existing repository from GitHub to your computer.

# Clone into a new folder named after the repo
git clone https://github.com/username/repo.git

# Clone into specific folder (e.g., current folder '.')
git clone https://github.com/username/repo.git .
10. .gitignore & Security

Create a file named .gitignore to prevent specific files (like environment variables, API keys, or node_modules) from being tracked.

# Example .gitignore content
.env
node_modules/
dist/

Note: Never push .env files containing API keys to a public repository.

11. Branching

Branching allows you to work on features or bug fixes in isolation without affecting the main codebase.

# List branches
git branch

# Create a new branch
git branch feature/login

# Delete a branch
git branch -d feature/login
12. Switching Branches (Checkout) 
# Switch to an existing branch
git checkout feature/login

# Create AND switch to a new branch (Shortcut)
git checkout -b feature/login

# Switch back to main
git checkout main
13. Merging & Pull Requests

Merging Locally

# Merge specific branch INTO current branch
git merge feature/login

Pull Requests (GitHub)

1. Push your feature branch: git push -u origin feature/login
2. Go to GitHub, click "Compare & Pull Request".
3. Review code and merge into main.

14. SSH Keys

Generate SSH keys to authenticate with GitHub without typing passwords.

# Generate a new SSH key
ssh-keygen -t rsa -b 4096 -C "email@example.com"

# Start SSH agent
eval "$(ssh-agent -s)"

# Add key to agent
ssh-add ~/.ssh/id_rsa

# Copy public key (to paste into GitHub Settings > SSH keys)
cat ~/.ssh/id_rsa.pub
15. CI/CD with Vercel

Deploying automatically when pushing to GitHub.

  1. Create account on Vercel.com.
  2. "Add New Project" and import from GitHub.
  3. Click Deploy.

Now, every time you git push to main, Vercel will rebuild and deploy your site automatically.
